# AdminService (Admin.Api)
## Назначение (продукт)
- Центральная точка управления для роли Admin.
- Управление отелями, пользователями/ролями, глобальными настройками.
- Агрегированные сводки по броням/заселениям.

## Функционал и API
- Управление пользователями/ролями (прокси к Identity) — назначение менеджеров на отели.
- Управление отелями и тарифами (прокси/композиция вызовов HotelService).
- Глобальные настройки: время удержания неоплаченной брони, параметры уведомлений.
- Сводки: агрегированный список отелей с числом броней/заездов на дату.

## Данные
- Минимальная таблица `SystemSettings` (Key, Value, UpdatedAt).
- Кешируемые сводки в памяти/redis (опционально).

## События (опционально)
- Может слушать `ReservationCreated/Cancelled` для обновления сводок.

## Нефункциональные требования
- Доступ только для роли `Admin`.
- Грейсфул деградация при недоступности нижележащих сервисов (таймауты, ретраи).
- Лимит скорости на массовые операции (опционально).

## DoD
- Авторизация проверяет роль Admin для всех эндпоинтов.
- Интеграционные тесты сводок с моками внешних сервисов.
- Таймауты/ретраи настроены; ошибки внешних сервисов мапятся в читаемые ответы.
- OpenAPI актуален; `.env.example` содержит строки подключений и настройки кэширования (если есть).
