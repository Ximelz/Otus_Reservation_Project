# Документы в каталоге Info
Это навигация по учебным материалам системы бронирования. Читайте сверху вниз: сначала общую архитектуру, затем детали по сервисам и шину сообщений.

## Что здесь есть
- `architecture.md` — основной архитектурный документ: бизнес-потоки, микросервисы, схемы БД, REST-сценарии, события, задания для студентов.
- `promt.md` — исходное техническое задание, по которому формировалась архитектура.
- `identity.md` — описание IdentityService: регистрация, JWT, роли, DoD.
- `customer.md` — CustomerService: профиль клиента, API me/поиск, валидация, DoD.
- `hotel.md` — HotelService: отели, номера, тарифы, сезонные цены, публичные GET, DoD.
- `reservation.md` — ReservationService: бронирования, защита от овербукинга, статусы, события, DoD.
- `admin.md` — AdminService: управление для Admin, сводки, настройки, DoD.
- `scheduler.md` — SchedulerService: планировщик напоминаний/NoShow, воркер, ретраи, DoD.
- `notification.md` — NotificationService: шаблоны, email/Telegram, очередь отправки, DoD.
- `gateway.md` — API Gateway (Ocelot): маршруты, проброс авторизации/корреляции, DoD.
- `rabbitmq.md` — шина RabbitMQ: топология, формат событий, подписка/публикация, полный перечень событий и подписчиков.
- `Архитектура проекта.bmp` — изображение (если нужно визуально).

## Как пользоваться
1) Начните с `architecture.md`, чтобы понять домен, роли, сервисы, БД и основные сценарии.
2) Выберите свой сервис и откройте соответствующий файл (`identity.md`, `customer.md`, `hotel.md`, `reservation.md`, `admin.md`, `scheduler.md`, `notification.md`, `gateway.md`):
   - продуктовая цель и функционал;
   - данные/события;
   - нефункциональные требования;
   - чек DoD для готовности.
3) Для интеграций и асинхронных взаимодействий используйте `rabbitmq.md`: там формат сообщений, топология exchange/queues, routing keys, список событий и кто их слушает.

## Рекомендации по работе
- Проверяйте DoD своего сервиса перед сдачей: миграции, OpenAPI, тесты, настройки окружения.
- Сверяйте контракты REST/событий с таблицами в `architecture.md` и `rabbitmq.md`, чтобы избежать рассинхронизации.
- Оставляйте заметки о допущениях и TODO прямо в файлах своего сервиса, если расширяете функциональность.
